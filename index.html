HTML

  • HTML stands for "hypertext markup language"
  • HTML defines the structure of a webpage
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • "Child elements" are enclosed by "parent elements"
  • An "element" is the whole statement of information that is contained by an opening and closing "tag," a direction enclosed in angle brackets.
  • Click here to see an HTML cheat sheet.

CSS

  • CSS defines the style (colors, fonts, etc.) of a webpage.
  • You can use CSS language within an HTML file in two ways: using inline commands or an internal style sheet.
  • Inline commands are included in the initial tag of the element you would like to style.
  • An internal style sheet is contained within the head of the HTML file, using the "style" tag.
  • Most commonly, however, an external style sheet is used.
  • An external style sheet (a .css file) can be linked to an HTML file by adding a "link" tag in the head.
  • By using a single common "class selector," a designated style can be applied to multiple elements within your HTML file.

Git

  • In short, Git is a version control system.
  • This allows users and teams to work together on projects, keeping track of the versions of and changes made to an application.
  • Tip: make sure to commit your work frequently
  • Tip: make sure you are working on the correct branch
  • Create a "pull request" to attempt to merge a branch you have worked on to the main repository (repo).
  • The basic workflow of GitHub is as follows: make changes on new branch, pull request (content reviewed), merge branches and deploy.
  • (You may at some point look into the application "Federalist" as part of your workflow)
  • A Git command always begins with "git"
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
    • Click here to see a Git command cheat sheet.

JavaScript

  • JavaScript allows interactivity with a webpage: effects, data communication, etc.
  • Control flow is the order in which a computer executes JS code. JS is read from top to bottom.
  • A "for" loop repeats a certain command until directed to stop.
  • Data types: string (text), number, Boolean (t/f), array, object (general).
  • A comment can be started with '/*' and ended with '*/'.
  • Conditional statement: tests if an expression returns true or not, exectues based on truth value.
  • Function: often followed by '()'. Needs to be called in order to work.
