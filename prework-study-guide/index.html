Prework Study Guide

HTML

  • HTML stands for "hypertext markup language"
  • HTML defines the structure of a webpage
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • "Child elements" are enclosed by "parent elements"
  • An "element" is the whole statement of information that is contained by an opening and closing "tag," a direction enclosed in angle brackets.

CSS

  • CSS defines the style (color, font, etc.) of a webpage.

Git

  • In short, Git is a version control system.
  • This allows users and teams to work together on projects, keeping track of the versions of and changes made to an application.
  • Tip: make sure to commit your work frequently
  • Tip: make sure you are working on the correct branch
  • Create a "pull request" to attempt to merge a branch you have worked on to the main repository (repo).
  • The basic workflow of GitHub is as follows: make changes on new branch, pull request (content reviewed), merge branches and deploy.
  • (You may at some point look into the application "Federalist" as part of your workflow)
  • A Git command always begins with "git"
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
